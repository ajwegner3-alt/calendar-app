# Milestone v1.1: Multi-User + Capacity + Branded UI

**Status:** ✅ SHIPPED 2026-04-30
**Phases:** 10-13 (with decimal phases 12.5 + 12.6 inserted between 12 and 13)
**Total Plans:** 34 plans across 6 phases
**Timeline:** 2026-04-27 → 2026-04-30 (3 days from kickoff to sign-off)

## Overview

v1.1 layered three tightly-scoped capability areas on top of the locked v1.0 foundation: (1) public multi-user signup + onboarding wizard with working email verification + password reset, (2) per-event-type booking capacity with race-safe DB enforcement that root-caused and replaced the v1.0 partial-unique-index pattern, and (3) a Cruip "Simple Light"-styled visual overhaul across all 5 owner-facing and public surfaces (dashboard, public booking page, embed widget, transactional emails, auth pages).

**Phase 12.5 + 12.6 inserted as decimal phases** between Phase 12 and Phase 13 — Phase 12.5 added per-account chrome theming + email token unification + FloatingHeaderPill removal (color-mix tinting approach); Phase 12.6 was a same-day course-correction by Andrew after Vercel review showed color-mix percentages produced visually-indistinguishable chromes. 12.6 replaced the tinting model with direct hex-color application across 3 controls (sidebar / page / primary) and wired `brand_primary` to the shadcn `--primary` CSS variable for the first time.

Phases executed strictly sequentially: **Phase 10 → Phase 11 → Phase 12 → Phase 12.5 → Phase 12.6 → Phase 13**. Within each phase, plan-level parallelization was enabled via `parallelization=true` (per `config.json`); plan-phase derived parallelizable plan groupings during planning.

Free tier — no Stripe in v1.1. v1.0 marathon QA carry-overs (EMAIL-08, QA-01..06) were RE-DEFERRED to v1.2 by project-owner discretion 2026-04-27 and confirmed at v1.1 sign-off.

### Phase 12.6: Direct Per-Account Color Controls (Page / Sidebar / Primary) (INSERTED)

**Goal**: A dashboard owner can independently configure THREE per-account colors and see each one render at full strength on the surface it controls, with auto-WCAG text contrast: sidebar color (sidebar bg), page background color (background_color column), and primary color (brand_primary wired to shadcn --primary CSS variable override). **Why 12.6**: Andrew reviewed Phase 12.5 on Vercel and found that at 6-14% color-mix percentages, even strong navy looked indistinguishable from gray-50. Phase 12.6 replaced the tinting model with direct hex application. `brand_primary` had also never been wired to the shadcn `--primary` variable, so buttons/switches/focus-rings were always black regardless of account branding — Phase 12.6 was the first wire-up.

**Plans**: 3 plans in 2 waves

**Code complete:** 2026-04-29 (Andrew live Vercel approval same day). 8 manual checks deferred to milestone-end QA.
