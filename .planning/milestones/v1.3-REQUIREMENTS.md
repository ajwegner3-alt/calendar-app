# Requirements: Calendar App v1.3 — Bug Fixes + Polish

**Defined:** 2026-05-02
**Core Value:** A visitor lands on a contractor's website, picks an available time slot in a branded widget, and walks away with a confirmed booking in their inbox — no phone tag, no back-and-forth.

**Milestone shape:** Surgical bug-fix and polish milestone surfaced from Andrew's post-v1.2 live use of the production app. No new architecture, no new packages, no new domain capabilities. All requirements targeted existing surfaces. REQ-IDs continue from v1.1+v1.2 category prefixes where applicable.

## v1.3 Requirements

8 requirements across 3 categories. Each mapped to a roadmap phase.

### Auth

- [x] **AUTH-18**: Clicking the "Sign up" link from `/login` navigates to `/signup` — SHIPPED (Phase 22 Plan 01; root cause: middleware `pathname.startsWith()` missed exempt paths; fix: exact-match `publicAuthPaths.includes()`)
- [x] **AUTH-19**: Sign-in page (`/login`) layout has the informational pane on the LEFT and the email/password form on the RIGHT — SHIPPED (Phase 22 Plan 01; column swap; preserved `lg:` breakpoint to avoid v1.2 regression)
- [x] **AUTH-20**: Authenticated owner sessions persist for 30 days via Supabase sliding refresh window — SHIPPED (Phase 22 Plan 02; verified `@supabase/ssr@0.10.2` 400-day cookie default not overridden; patched `proxy.ts setAll` to forward cache-control headers; tracked observationally over week-of-use)

### Public Booking

- [x] **PUB-13**: On mobile viewports (< 768px) the slot picker calendar widget renders horizontally centered within the public booking card — SHIPPED (Phase 23 Plan 02; `mx-auto` initial fix failed mobile live test; corrected to `justify-self-center` (canonical grid-item alignment) in commit `23973c9`)
- [x] **PUB-14**: On desktop viewports the timezone hint and "Pick a date to see available times:" instruction do not visually overlap the calendar widget — SHIPPED (Phase 23 Plan 02; timezone hint hoisted above grid wrapper as full-width sibling via React fragment root)
- [x] **PUB-15**: The public account index page (`/[account]`) lists every public event type for that account as a selectable card with logo / event name / duration / CTA — SHIPPED (Phase 23 Plan 01; metadata-only fix; event-type cards listing UI was already shipped in a prior phase; only browser title was missing — `generateMetadata()` now returns `Book with ${name}`)

### Owner UI

- [x] **OWNER-12**: The Owner Home tab monthly calendar (the day-grid on `/app/home`) does NOT render orange (`#F97316` accent) on day buttons in any state — SHIPPED (Phase 24 Plan 01; per-instance className overrides on `home-calendar.tsx` DayButton; selected = `bg-gray-700`, today = `bg-muted` + grey ring, has-bookings dot = `#9CA3AF`; `globals.css --color-accent` token preserved for 3 other consumers)
- [x] **OWNER-13**: The event-type edit page (`/app/event-types/[id]`) renders a copyable booking-link field at the top of the form showing the public per-event URL with a copy-to-clipboard button — SHIPPED (Phase 24 Plan 02; new `BookingLinkField` client component as first form section, live-updates from `watch("slug")`, Copy → Check icon flip ~1.5s; mid-checkpoint scope expansion added per-row `RowCopyLinkButton` on event-types list page + blue per-instance focus override on `DropdownMenuItem`s; `UrlPreview` deleted)

## Out of Scope

Explicit boundaries — documented to prevent scope creep INTO v1.3.

| Feature | Reason |
|---------|--------|
| Stripe / billing / monetization | Distribution-first posture; revenue layer reserved for a separate milestone after multi-user proves out |
| Google Calendar / iCal / Outlook two-way sync | Andrew explicitly wants Supabase as sole source of truth — no external calendar OAuth |
| Custom subdomains (`book.clientsite.com`) | Path-based URLs sufficient |
| Custom CSS white-label | Logo + brand color suffices |
| Configurable reminder timing | Hardcoded 24h |
| Per-event-type availability schedules | Account-wide availability |
| Multiple reminders (24h + 1h) | Single 24h reminder |
| SMS notifications | Email only |
| Mobile app | Web-only |
| Round-robin / team scheduling | Anti-feature |
| Workflow builder | Anti-feature — n8n exists for this |
| Video conferencing integration | Trade bookings are in-person |
| Recurring bookings | Trade bookings are one-off jobs |
| Waitlists / Group bookings | Demand not justified in this vertical |
| Two-way SMS chat | Pushes into messaging-platform territory |
| Temporal (JS proposal) | v1 uses `date-fns v4 + @date-fns/tz` |
