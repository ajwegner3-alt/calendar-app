### Phase 27: Slot Correctness (DB-Layer Enforcement)

**Goal:** Close the contractor-can't-be-in-two-places-at-once invariant at the database layer by adding an account-scoped EXCLUDE constraint that rejects overlapping bookings across different event types.

**Depends on:** None. Independent of Phases 25 and 26. Internal task sequence is fixed (pre-flight → migration → error mapping → tests → smoke).

**Plans:** 3 plans

Plans:

- [x] 27-01: Pre-flight diagnostic (HARD GATE) + EXCLUDE constraint migration (btree_gist + during generated column + constraint, with rollback SQL) — completed 2026-05-03
- [x] 27-02: Application-layer 23P01 error mapping (route.ts + reschedule.ts) + booker UI 409 handler (CROSS_EVENT_CONFLICT, generic copy) — completed 2026-05-03
- [x] 27-03: 6 mandatory tests (cross-event block, group-booking regression, adjacent non-collision, cancelled-doesn't-block, reschedule cross-event, retry-loop-break) + Andrew production smoke runbook — completed 2026-05-03

**Details:**

- Pre-flight diagnostic returned 0 cross-event overlap rows on a 6-row bookings table. V14-CP-06 hard gate satisfied without manual data resolution. Single-step `ADD CONSTRAINT` chosen (well under 10k threshold; NOT VALID + VALIDATE branch deleted from file).
- Constraint definition (verified verbatim against production via `pg_constraint`): `EXCLUDE USING gist (account_id WITH =, event_type_id WITH <>, during WITH &&) WHERE ((status = 'confirmed'::booking_status))`.
- Generated column (verified via `information_schema.columns`): `during tstzrange GENERATED ALWAYS AS (tstzrange(start_at, end_at, '[)'::text)) STORED`. Half-open `[)` bound prevents adjacent-slot false collisions (V14-CP-02).
- `btree_gist` extension installed first (V14-CP-01). Rollback file does NOT drop the extension (harmless to keep, may be reused).
- 23P01 error mapping in `app/api/bookings/route.ts`: new branch placed BEFORE the `code !== "23505"` check in the retry loop, breaks immediately without incrementing `slot_index` (V14-MP-01). Post-loop block has 23P01 branch BEFORE the 23505 branch, returning 409 with body `{error, code: "CROSS_EVENT_CONFLICT"}`.
- 23P01 error mapping in `lib/bookings/reschedule.ts`: maps to existing `reason: "slot_taken"` (NOT a new reason) — reuses `/api/reschedule` 409 SLOT_TAKEN response unchanged. `app/api/reschedule/route.ts` deliberately UNTOUCHED.
- Booker UI 409 handler (`booking-form.tsx`) treats CROSS_EVENT_CONFLICT identically to SLOT_TAKEN — same generic banner copy, same `onRaceLoss` auto-refresh-and-retry, same Turnstile reset. No event-type leak in the message string.
- Observability: `[/api/bookings] 23P01 cross-event overlap` logs `{ code, account_id, event_type_id }`; `[reschedule] 23P01 cross-event overlap` logs `{ code, booking_id }`. NO PII (booker_email, booker_name, booker_phone, ip absent).
- 6-test pinning suite at `tests/cross-event-overlap.test.ts`: cross-event block, group-booking regression, adjacent non-collision, cancelled-doesn't-block, reschedule cross-event, retry-loop-break. Tests 1-5 use `describe.skipIf(skipIfNoDirectUrl)` (V14-MP-05); Test 6 is a static-text scan placed OUTSIDE the skipIf so it runs in CI without `SUPABASE_DIRECT_URL`.
- Andrew live-verified Phase A of the smoke runbook on production `nsi` account: booked 10:00–10:30 on 2026-05-07; cross-event re-book at 10:00 correctly rejected. Phase B (raw curl) Turnstile-blocked; acceptable per plan's explicit `partial: A passed, B blocked by Turnstile` allowance. Buffer-related observation about the 10:30 slot resolved as pre-existing v1.0 `accounts.buffer_minutes=15` on nsi account (`lib/slots.ts:203` `slotConflictsWithBookings`), unmodified by Phase 27. Andrew chose option (a) keep buffer behavior.

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- **EXCLUDE constraint mechanism (Option A) over trigger (Option B)** — DDL live-verified against production Postgres 17.6.1 by research. Trigger is fallback only.
- **Pre-flight diagnostic as hard gate (V14-CP-06)** — `VALIDATE CONSTRAINT` aborts on existing cross-event overlapping confirmed bookings, leaving migration in unrecoverable partial state. Pre-flight runs and confirms zero rows BEFORE any DDL.
- **23P01 error mapping locked in two files** — `app/api/bookings/route.ts` (V14-MP-01: branch BEFORE 23505 check, `break` without `slot_index` increment) and `lib/bookings/reschedule.ts` (V14-MP-02: maps to existing `slot_taken` reason).
- **No new reschedule reason** — 23P01 maps to existing `slot_taken` so `/api/reschedule/route.ts` continues returning its existing 409 SLOT_TAKEN body unchanged. CONTEXT-locked.
- **Generic booker UX (no event-type leak)** — CROSS_EVENT_CONFLICT message is `"That time is no longer available. Please choose a different time."` Identical wording to the booking-form's SLOT_TAKEN fallback. No "another appointment" or "event type" language.
- **PII-free observability logs** — only structural identifiers (`code`, `account_id`, `event_type_id`, `booking_id`); never `booker_email`, `booker_name`, `booker_phone`, or `ip`.
- **Migration apply path locked** — `echo | npx supabase db query --linked -f <file>`. `supabase db push --linked` is broken in this repo.
- **Static-text scan tests for control-flow invariants** — Phase 26 precedent extended in Phase 27 Test 6 (`retry-loop-break`). Read source file as `fs` text, regex-assert structural property. Zero new dependencies. Placed OUTSIDE `describe.skipIf` so it runs in CI without `SUPABASE_DIRECT_URL`.
- **`describe.skipIf(skipIfNoDirectUrl)` pattern (V14-MP-05)** — pg-driver tests skip cleanly in CI. Mirrors `tests/race-guard.test.ts` precedent.
- **Per-instance className override pattern extended to Phase 25** — shared `components/ui/calendar.tsx` and `globals.css --color-accent` UNTOUCHED (preserves Phase 23/24 invariant). All v1.4 calendar styling done at the consumer site (`home-calendar.tsx:72`).
- **`buffer_minutes` is account-scoped and out of scope for Phase 27** — the `nsi` production account has `buffer_minutes = 15`, which causes `lib/slots.ts:203` `slotConflictsWithBookings` to pre-hide adjacent slots. Pre-existing v1.0 behavior; unmodified by Phase 27. The DB constraint correctly allows `[)` adjacency at the constraint level (Test 3 pins this); the picker hides slots earlier due to buffer. Andrew chose option (a) keep buffer behavior.
- **Marathon QA waived (4th consecutive milestone)** — pattern formally adopted as deploy-and-eyeball at v1.3 close; v1.4 followed the same model. Andrew approved each phase live as it shipped.
