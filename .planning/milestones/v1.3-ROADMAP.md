## Milestone Summary

**Decimal Phases:**

None — v1.3 used only sequential phases (22, 23, 24). No INSERTED phases this milestone.

**Key Decisions:**

- **Marathon QA formally adopted as deploy-and-eyeball** (third deferral now permanent) — Andrew explicitly chose surgical bug-fix scope over scheduling QA-09..QA-13. Carryover backlog (5 clusters: Marathon QA, Resend/Vercel Pro, OAuth/magic-link, NSI mark, tech debt) deferred to v1.4 in entirety. Pattern locked: live deploy approval is the ship gate.
- **Per-instance className overrides over shared-component edits** (Phase 23 invariant extended) — `slot-picker.tsx` `mx-auto` → `justify-self-center`, `home-calendar.tsx` `bg-gray-700`/`#9CA3AF`, `row-actions-menu.tsx` `focus:bg-primary`. Shared `components/ui/calendar.tsx` and `components/ui/dropdown-menu.tsx` stayed untouched across all 3 phases. Pattern locked for any future shared-component theming need where blast-radius matters.
- **`globals.css --color-accent` token preserved** — 3 other consumers depend on it (public booker `.day-has-slots::after`, bookings-table hover, cancel-confirm-form hover). De-orange happened via instance overrides, not token redefinition.
- **`@supabase/ssr@0.10.2` 400-day default trusted** — no `cookieOptions.maxAge` override; any custom value would actively shorten sessions. ROADMAP success criterion #3 (close-browser-reopen-next-day) tracked observationally over week-of-use, not blocking plan close.
- **`current_owner_account_ids` RPC + `accounts.slug` query LOCKED as canonical owner-context lookup** — Phase 24 Plan 02 third site to use this pattern (after `event-types/page.tsx` and `loadBrandingForOwner`). Required-prop typing on `EventTypeForm.accountSlug` forced `new/page.tsx` conversion to async server component, surfacing the pattern at the create-page consumer too.
- **Mid-checkpoint user-driven scope expansion accepted** — Phase 24 Plan 02 absorbed two follow-up requirements during the human-verify checkpoint (`RowCopyLinkButton` for the list page, blue dropdown highlight) into commit `db7fb62`. Pattern: mid-execution feedback that fits the same surface gets bundled rather than spawning new plans.
- **Same-day milestone is viable for surgical scope** — full lifecycle (scope lock → requirements → roadmap → 3 phases planned → 3 phases executed → verifier passes → milestone close) compressed to ~10 hours. Pattern: when scope is < 10 plans and all touch existing surfaces, a single working day suffices.

**Issues Resolved:**

- AUTH-18: middleware redirect loop on `/app/signup` (link click did nothing) — root cause: `pathname.startsWith()` check missed exempt paths. Fixed via exact-match `publicAuthPaths.includes()`.
- AUTH-19: login page split-panel reversed from intended Cruip direction — fixed via column swap (lg breakpoint preserved).
- AUTH-20: session re-prompted too frequently — root cause: dashboard timebox + inactivity were already 0; verified `@supabase/ssr` 400-day cookie default not overridden anywhere. Patched `proxy.ts setAll` to forward cache-control headers (defensive against CDN cache poisoning during token rotation).
- PUB-13: mobile calendar off-center inside `w-fit` shadcn Calendar root — fixed first via `mx-auto` (failed live mobile test) then via `justify-self-center` (canonical grid-item alignment property; worked on Andrew's live deploy).
- PUB-14: timezone hint + "Pick a date" instruction overlapped calendar widget at desktop — fixed by hoisting timezone hint above grid wrapper as full-width sibling via React fragment root.
- PUB-15: bare `/[account]` landing missing browser title — fixed via `generateMetadata()` returning `Book with ${name}` (event-type cards listing UI was already shipped in a prior phase; only metadata was missing).
- OWNER-12: orange `#F97316` (`--color-accent`) leaking through `/app/home` day-button hover state — fixed by replacing 3 className blocks (hover, selected, today) + 1 inline-style (booking dot) with grey-only equivalents. `globals.css` token preserved.
- OWNER-13: `/app/event-types/[id]/edit` had only a non-copyable `yoursite.com/nsi/[slug]` placeholder — replaced with new `BookingLinkField` client component rendering as first form section (live-updates from `watch("slug")`). `UrlPreview` deleted entirely.
