**Step 1 — Confirm `bookings_no_double_book` index exists:**
```sql
SELECT indexname, indexdef, pg_size_pretty(pg_relation_size(indexname::regclass)) AS size
FROM pg_indexes
WHERE tablename = 'bookings'
  AND indexname IN ('bookings_no_double_book', 'bookings_capacity_slot_idx');
```

**Step 2 — Find any (event_type_id, start_at) pair with >1 confirmed bookings:**
```sql
SELECT event_type_id, start_at, count(*) AS duplicates,
       array_agg(id ORDER BY created_at) AS booking_ids
FROM bookings
WHERE status = 'confirmed'
GROUP BY event_type_id, start_at
HAVING count(*) > 1;
```

**Step 3 — Confirm `start_at` is `timestamptz` (not text or differing precision):**
```sql
SELECT column_name, data_type, datetime_precision
FROM information_schema.columns
WHERE table_name = 'bookings' AND column_name = 'start_at';
```

**Step 4 — If Step 2 returned rows, pull lifecycle audit for the conflicting bookings:**
```sql
-- Substitute <event_type_id> and <start_at> with values from Step 2 results.

       be.event_type AS audit_event, be.occurred_at, be.metadata
```

**Step 5 — Quantify the rescheduled-status slot reuse gap (Pitfall 4 hypothesis):**
```sql
-- Are there bookings where status='rescheduled' shares (event_type_id, start_at) with a 'confirmed' booking?
-- If yes: this is the smoking-gun root-cause hypothesis (RESEARCH §CAP-01 primary hypothesis).
SELECT b1.event_type_id, b1.start_at,
       b1.id AS rescheduled_id, b1.status AS rescheduled_status,
       b2.id AS confirmed_id, b2.status AS confirmed_status
FROM bookings b1
JOIN bookings b2
  ON b1.event_type_id = b2.event_type_id
  AND b1.start_at = b2.start_at
  AND b1.id != b2.id
WHERE b1.status = 'rescheduled' AND b2.status = 'confirmed';
```

**Step 6 — Check microsecond precision drift (timestamp values that LOOK identical but differ by sub-second):**
```sql
-- Only run if Step 2 found duplicates. Compares to_char microseconds.
-- Substitute values from Step 2.
SELECT id, start_at::text AS start_at_text,
       to_char(start_at, 'YYYY-MM-DD HH24:MI:SS.US TZ') AS precise,
       extract(epoch from start_at) AS epoch_seconds, status
FROM bookings
WHERE event_type_id = '<from-step-2>'
  AND date_trunc('minute', start_at) = date_trunc('minute', '<from-step-2>'::timestamptz)
ORDER BY start_at;
```
