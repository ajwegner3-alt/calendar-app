## Decisions Made

- **resolveChromeColors returns primaryColor as string (never null):** `Branding.primaryColor` always has a `DEFAULT_BRAND_PRIMARY` fallback; Wave 2 consumers set `--primary` unconditionally which is safe because the default value matches the existing shadcn override. v1.2 improvement: store raw nullable brand_primary separately so the override can be skipped when owner hasn't set it.
- **chromeTintToCss kept as live export (not deprecated alias):** 12.5 shell layout and app-sidebar still call `chromeTintToCss`; keeping it live avoids breaking those callers before 12.6-02 replaces them. The plan said "compat export" — implemented as the full original function, which satisfies the intent.
- **sidebar_color kept as separate column from background_color:** Planner locked this — rename is risky on prod with existing data. `background_color` continues serving semantically as the page color field without a DB rename.
