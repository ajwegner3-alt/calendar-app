Simplify `renderEmailBrandedHeader` color resolution from `sidebarColor ?? brand_primary ?? DEFAULT` to `brand_primary ?? DEFAULT`. Update 2 test files. Single atomic commit, single Vercel deploy, single live smoke test.

Purpose: Phase 19 is the email-side counterpart to Phase 18's web-side branding collapse. After Phase 18 narrowed the `Branding` web type, Phase 19 narrows the `EmailBranding` email type. Both phases together strip every runtime read of `sidebar_color`, `background_color`, `chrome_tint_intensity` so Phase 21 can drop those columns safely.

Output: 12 modified files, single git commit, deploy log green, confirmation email observed live with `#3B82F6` header band + text-only footer.

3. CSP lives only in `proxy.ts`, never `next.config.ts`.
4. Two-stage owner auth — RLS pre-check before service-role mutation.
5. `background_shade` ENUM must be dropped alongside the column (Phase 21 territory; nothing to do here).
6. DROP migration = two-step deploy (Phase 21).

**Atomic-deploy lock (CP-02):** All 12 files ship in ONE commit, ONE deploy. The narrowed `EmailBranding` interface immediately compiler-errors any caller still passing deprecated fields — this is the design. Do NOT split into multiple commits.
