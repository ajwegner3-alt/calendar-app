## Decisions Made

- **Tinting strategy locked:** `color-mix(in oklch, ${color} N%, white)` — same primitive as `shadeToGradient` flatTint. No opacity overlays, no pre-computed token tables. Composable with existing gradient backdrop.
- **Percentages locked in TINT_PCT table:** sidebar full=14%, subtle=6%; page full=8%, subtle=3%. Cards always white (preserves Cruip card-on-tint readability). Do not fork this table in Wave 2 consumers.
- **null propagation pattern confirmed:** `chromeTintToCss` returns null for `intensity='none'` or `color=null`. Consumers should apply via `style={{ backgroundColor: chromeTintToCss(...) ?? undefined }}` so null falls through to CSS class default.
- **pickTextColor required no changes:** Already exported from `lib/branding/contrast.ts` since Plan 12-01. `chrome-tint.ts` imports directly from there. BRAND-09 DRY requirement satisfied without code movement.
- **chromeTintTextColor uses original color luminance as WCAG proxy:** Conservative but correct for typical brand colors. A dark navy at 14% into white produces a light surface (black text should be correct) but `pickTextColor('#0A2540')` returns white because the original color is dark — this is a known trade-off documented in the helper's JSDoc. Wave 2 can refine if needed.
