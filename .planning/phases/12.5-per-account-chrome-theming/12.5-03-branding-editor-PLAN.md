<objective>
Extend the /app/branding editor with a chrome intensity picker and update the mini-preview
card to render a chrome-aware live preview showing how the sidebar and page bg will look.

Purpose: Owners need a way to set chrome_tint_intensity. This plan wires the full round-trip:
DB schema → BrandingState loader → Zod schema → saveBrandingAction → intensity picker UI
→ live mini-preview. The mini-preview is the only in-page chrome preview (CONTEXT.md lock);
owners navigate to the dashboard to see the full in-context experience.

Architectural decisions committed:
- MiniPreviewCard layout: horizontal split showing a narrow faux-sidebar strip on the left
  (tinted per sidebar intensity) and a tinted page area on the right (tinted per page intensity).
  Inside the page area, a white faux-card sits to illustrate the card-stays-white invariant.
  GradientBackdrop circles sit INSIDE the page area behind everything (composited on top of
  the page bg tint, same as the real dashboard).
- Intensity picker UI: three buttons (None / Subtle / Full) matching the ShadePicker component
  pattern from Plan 12-01. A new IntensityPicker component at:
  `app/(shell)/app/branding/_components/intensity-picker.tsx`
- Save lifecycle: chromeTintIntensity is saved alongside backgroundColor and backgroundShade
  in a single saveBrandingAction call (no separate save button — fold into the existing
  "Save background" button so the owner saves all three in one click).

Output: Updated schema.ts, actions.ts, load-branding.ts, branding-editor.tsx,
mini-preview-card.tsx, and new intensity-picker.tsx.
</objective>

Implementation:
```typescript
"use client";
import type { BackgroundShade, ChromeTintIntensity } from "@/lib/branding/types";
import { GradientBackdrop } from "@/app/_components/gradient-backdrop";
import { chromeTintToCss } from "@/lib/branding/chrome-tint";

interface MiniPreviewCardProps {
  color: string | null;
  shade: BackgroundShade;
  chromeTintIntensity: ChromeTintIntensity;
}

export function MiniPreviewCard({ color, shade, chromeTintIntensity }: MiniPreviewCardProps) {
  const sidebarBg = chromeTintToCss(color, chromeTintIntensity, "sidebar");
  const pageBg = chromeTintToCss(color, chromeTintIntensity, "page");

  return (
    <div className="space-y-1.5">
      <p className="text-sm font-medium text-muted-foreground">Preview</p>
      <div
        className="relative h-48 overflow-hidden rounded-lg border"
        style={{ backgroundColor: pageBg ?? undefined }}
      >
        {/* Faux sidebar strip */}
        <div
          className="absolute left-0 top-0 bottom-0 w-12 border-r border-border/30"
          style={{ backgroundColor: sidebarBg ?? "hsl(var(--sidebar))" }}
        >
          {/* Sidebar shimmer items */}
          <div className="flex flex-col gap-2 p-2 pt-3">
            {[1, 2, 3, 4].map((i) => (
              <div key={i} className="h-2 rounded-full bg-foreground/15 w-full" />
            ))}
          </div>
        </div>

        {/* Page area */}
        <div className="relative ml-12 h-full overflow-hidden">
          {/* GradientBackdrop composited on top of page tint */}
          <GradientBackdrop color={color} shade={shade} />

          {/* Faux card — always white, invariant of intensity */}
          <div className="relative mx-3 mt-4 rounded-md bg-white p-3 shadow-sm">
            <div className="h-2.5 w-2/3 rounded-full bg-foreground/10 mb-2" />
            <div className="h-2 w-1/2 rounded-full bg-foreground/8 mb-1" />
            <div className="mt-2 flex gap-1.5">
              <div className="h-6 w-14 rounded bg-foreground/15" />
              <div className="h-6 w-10 rounded bg-foreground/8" />
            </div>
          </div>
        </div>
      </div>
    </div>
  );
}
```

Note on Phase 7 pitfall: ALL runtime colors (sidebarBg, pageBg) use inline `style` props.
`hsl(var(--sidebar))` is a static CSS token string — valid inline style. No dynamic Tailwind.
