`md:hidden` means mobile-only. Desktop sidebar is always open; trigger is redundant there.

### Page Bg Tinting Pattern
```tsx
const pageBgTint = chromeTintToCss(branding.backgroundColor, branding.chromeTintIntensity, "page");
// ...
<SidebarInset
  className="relative overflow-hidden bg-background"
  style={{ backgroundColor: pageBgTint ?? undefined }}
>
```
When pageBgTint is null (intensity='none' or no color), `?? undefined` removes the style attribute and `bg-background` (gray-50) applies — Phase 12 regression-safe baseline.

### Sidebar Tinting + WCAG Text Flip
```tsx
const sidebarBgTint = chromeTintToCss(backgroundColor, chromeTintIntensity, "sidebar");
const sidebarTextColor = chromeTintTextColor(backgroundColor, chromeTintIntensity, "sidebar");

<Sidebar
  collapsible="icon"
  style={{
    backgroundColor: sidebarBgTint ?? undefined,
    ...(sidebarTextColor
      ? ({ "--sidebar-foreground": sidebarTextColor } as React.CSSProperties)
      : {}),
  }}
>
```
`--sidebar-foreground` CSS variable overrides text/icon color for all nav items via the existing Tailwind CSS var chain — no per-item inline styles needed.

### Regression-Safe Path (intensity='none')
- `chromeTintToCss` returns null → `?? undefined` → inline style absent → CSS token (`--sidebar`, `bg-background`) applies
- Visually identical to Phase 12 gray-50 baseline

## Decisions Made
- **WCAG via CSS variable, not per-item styles** — overriding `--sidebar-foreground` on the Sidebar root propagates to all nav buttons automatically via shadcn's CSS var chain. Cleaner than applying inline style to each SidebarMenuButton.
- **`md:hidden` on hamburger** — desktop sidebar always visible; trigger only needed on mobile where sidebar is a drawer.
- **Slimmed accounts SELECT** — `name` and `logo_url` were exclusively used by FloatingHeaderPill; removed to keep the query lean.
- **Cards not tinted** — locked in 12.5-CONTEXT.md; Cruip card-on-tint readability requires white card surfaces.
