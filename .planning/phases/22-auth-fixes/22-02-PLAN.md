4. **One small correctness fix exists in code.** The `setAll` callback in `lib/supabase/proxy.ts` currently has signature `setAll(cookiesToSet)` but `@supabase/ssr@0.10.2`'s `SetAllCookies` type passes a second `headers: Record<string, string>` argument carrying cache-control directives. Omitting it means cache headers from token-refresh responses aren't forwarded to the response, which can cause CDN cache issues on rotation. Patching the signature to forward headers is a low-risk correctness fix that aligns with current `@supabase/ssr` behavior.

  <action>
Edit `lib/supabase/proxy.ts` to update the `setAll` callback signature to accept and forward the `headers` parameter that `@supabase/ssr@0.10.2` now passes.

Current code (lines 27–35):
```ts
setAll(cookiesToSet) {
  cookiesToSet.forEach(({ name, value }) =>
    request.cookies.set(name, value),
  );
  supabaseResponse = NextResponse.next({ request });
  cookiesToSet.forEach(({ name, value, options }) =>
    supabaseResponse.cookies.set(name, value, options),
  );
},
```

Replace with:
```ts
setAll(cookiesToSet, headers) {
  cookiesToSet.forEach(({ name, value }) =>
    request.cookies.set(name, value),
  );
  supabaseResponse = NextResponse.next({ request });
  cookiesToSet.forEach(({ name, value, options }) =>
    supabaseResponse.cookies.set(name, value, options),
  );
  // AUTH-20 v1.3: forward cache-control headers from @supabase/ssr@0.10.2's
  // SetAllCookies callback. Without this, cache directives emitted during
  // token refresh aren't propagated to the response, which can cause CDN
  // cache poisoning on rotation. The `headers` argument is part of the
  // current SetAllCookies type signature (see node_modules/@supabase/ssr
  // /dist/main/types.d.ts).
  Object.entries(headers ?? {}).forEach(([key, value]) =>
    supabaseResponse.headers.set(key, value),
  );
},
```
