**DB-04 satisfaction note:** Phase 11-03's reference migration uses `DO $$ BEGIN ... END $$;` for a *runtime SQL precondition check* (verifying an index existed before dropping the old one). Phase 21's analog precondition (CP-01 grep) cannot be expressed in SQL — grep is a filesystem operation. The plan satisfies DB-04's defensive intent via:
1. `IF EXISTS` guards on every drop (idempotent, safe re-run)
2. `BEGIN/COMMIT` transactional wrap (all-or-nothing atomicity)
3. `DO $$ BEGIN RAISE NOTICE '...'; END $$;` header for log clarity (matches Phase 11-03 syntactic style)
4. The substantive defensive check (CP-01 grep + tsc + ENUM verification) lives in Task 1 pre-flight gates, NOT in SQL

**Ordering note (CRITICAL):** ALL `DROP COLUMN` BEFORE all `DROP TYPE`. Postgres rejects `DROP TYPE` if any column still references the type (`cannot drop type X because other objects depend on it`). Inside one `BEGIN/COMMIT` block this is safe — partial failure rolls back atomically.
