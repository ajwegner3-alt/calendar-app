    ```typescript
    // lib/stripe/prices.ts
    // Centralized pricing config. Reads price IDs and amounts from env vars so PREREQ-E
    // can finalize values without code changes. The webhook writes plan_interval from
    // Stripe's payload — this file is for UI display + Checkout session creation only.

    const monthlyCents = parseInt(process.env.STRIPE_PRICE_MONTHLY_CENTS ?? "2900", 10);
    const annualCents = parseInt(process.env.STRIPE_PRICE_ANNUAL_CENTS ?? "27840", 10);

    function dollars(cents: number): string {
      return `$${(cents / 100).toFixed(cents % 100 === 0 ? 0 : 2)}`;
    }

    export const PRICES = {
      monthly: {
        priceId: process.env.STRIPE_PRICE_ID_MONTHLY ?? "price_placeholder_monthly",
        amountCents: monthlyCents,
        label: `${dollars(monthlyCents)}/month`,
      },
      annual: {
        priceId: process.env.STRIPE_PRICE_ID_ANNUAL ?? "price_placeholder_annual",
        amountCents: annualCents,
        // total billed once a year, e.g. "$232/year"
        totalLabel: `${dollars(annualCents)}/year`,
        // monthly equivalent for headline display, e.g. "$19.33/month"
        monthlyEquivalentLabel: `${dollars(Math.round(annualCents / 12))}/month`,
        // computed dynamically per CONTEXT lock — do NOT hardcode 20%
        savingsPct: Math.max(
          0,
          Math.round((1 - annualCents / (monthlyCents * 12)) * 100),
        ),
      },
    } as const;

    export type PriceInterval = "monthly" | "annual";
    ```

  <action>
    Create `app/api/stripe/checkout/route.ts` implementing the full pattern from RESEARCH §Code Examples (Full Checkout Route Handler skeleton). Key requirements:

    1. **Module setup:**
       - `import "server-only";`
       - `export const runtime = "nodejs";`
       - `export const dynamic = "force-dynamic";`
       - Import: `NextResponse` from `next/server`, `createClient` from `@/lib/supabase/server`, `createAdminClient` from `@/lib/supabase/admin`, `stripe` from `@/lib/stripe/client`, `PRICES, type PriceInterval` from `@/lib/stripe/prices`.

    2. **Auth gate:** Use `createClient()` + `await supabase.auth.getClaims()`. If no claims, return `401 {error:"unauthorized"}` with `Cache-Control: no-store`.

    3. **Body parse:** `const body = await req.json().catch(() => ({}))`. Coerce `body.interval` to `PriceInterval` — default to `"monthly"` if not exactly `"annual"`. Resolve `priceId` from `PRICES[interval].priceId`. Reject with `400 {error:"missing_price_id"}` if `priceId.startsWith("price_placeholder_")` AND `process.env.NODE_ENV === "production"` (safety net so a misconfigured prod env can't create a broken session).

    4. **Account fetch:** From `accounts` (RLS-scoped via server client) select `id, owner_email, stripe_customer_id` filtered `is("deleted_at", null) .maybeSingle()`. Return `404 {error:"account_not_found"}` if missing.

    5. **Customer upsert (SC-5 closure — see RESEARCH Pitfall 1 + 2):**
       - If `account.stripe_customer_id` exists, reuse it.
       - Else: `stripe.customers.create({ email: account.owner_email ?? undefined, metadata: { account_id: account.id } })`.
       - Switch to admin client. Conditional update: `admin.from("accounts").update({ stripe_customer_id: customer.id }).eq("id", account.id).is("stripe_customer_id", null)`.
       - If the conditional update returned an error (race lost), re-fetch `stripe_customer_id` and use the winner. Do NOT delete the orphaned customer (Phase 44 cleanup concern; out of scope).

    6. **Origin derivation** (mirror the pattern from `app/api/bookings/route.ts` lines ~85-88):
       ```typescript
       const origin =
         process.env.NEXT_PUBLIC_APP_URL ??
         (process.env.NEXT_PUBLIC_VERCEL_URL
           ? `https://${process.env.NEXT_PUBLIC_VERCEL_URL}`
           : "http://localhost:3000");
       ```

    7. **Session create:**
       ```typescript
       const session = await stripe.checkout.sessions.create({
         mode: "subscription",
         customer: customerId,
         client_reference_id: account.id, // belt-and-suspenders for webhook (used by 42-02)
         line_items: [{ price: priceId, quantity: 1 }],
         success_url: `${origin}/app/billing?session_id={CHECKOUT_SESSION_ID}`,
         cancel_url: `${origin}/app/billing`,
       });
       ```

       MUST NOT pass `subscription_data.trial_end` (RESEARCH Pitfall 9 — trial is app-side only).
       MUST NOT pass `customer_creation` (invalid in subscription mode — RESEARCH State of the Art).
       MUST NOT write `subscription_status` or `plan_interval` from this route (LD-10 + RESEARCH Pitfall 8 — webhook is canonical).

    8. **session.url validation** (RESEARCH Pitfall 3): If `!session.url`, return `500 {error:"no_session_url"}`.

    9. **Logging:** On every outcome (success or non-2xx) `console.log("[stripe-checkout] ...", {account_id, customer_id, interval, outcome})`. Match the structured-log style of the webhook handler.

    10. **All responses** must include `{ headers: { "Cache-Control": "no-store" } }`.
  </action>

    Create `app/api/stripe/checkout/status/route.ts` per RESEARCH Pattern 5:

    1. **Module setup** (same as Task 2): `"server-only"`, `runtime = "nodejs"`, `dynamic = "force-dynamic"`.

    2. **GET handler** (no body, no params — derives identity from auth cookie):
       - Auth gate via `createClient()` + `auth.getClaims()`. Return `401 {error:"unauthorized"}` with `Cache-Control: no-store` if missing.
       - Query: `supabase.from("accounts").select("subscription_status").is("deleted_at", null).maybeSingle()`.
       - Return `NextResponse.json({ subscription_status: account?.subscription_status ?? null }, { headers: { "Cache-Control": "no-store" } })`.
       - On DB error, return `500 {error:"db_error", subscription_status: null}` (still with no-store header — client treats transient errors as "keep polling").

    3. **CRITICAL — RESEARCH Pitfall 4 (cache-busting):**
       - `export const dynamic = "force-dynamic"` is mandatory.
       - `Cache-Control: no-store` on every response is mandatory.
       - Without both, the poller will see stale `trialing` forever.

    4. **Do NOT include `stripe_customer_id`, `current_period_end`, or any other column.** This endpoint is single-purpose: tell the poller whether the webhook has flipped status to `'active'`.

    5. **Do NOT add a `session_id` query-param check.** The auth session identifies the account; the `session_id` in the URL exists for forensic/UX framing only.
  </action>
