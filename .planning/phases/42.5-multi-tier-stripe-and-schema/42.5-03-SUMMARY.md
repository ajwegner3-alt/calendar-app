# Phase 42.5 Plan 03: Checkout Route Refactor Summary

**Refactors `POST /api/stripe/checkout` to accept `{tier, interval}` body shape, rejecting `tier='branding'` with `400 {error:'use_consult_link'}` and unknown/missing tier with `400 {error:'unknown_tier'}`, while preserving every other aspect of the route (auth gate, account lookup, customer upsert race guard, success/cancel URLs, return shape) per LD-18.**

## Accomplishments

- `app/api/stripe/checkout/route.ts` now imports `{getPriceId, PriceTier, PriceInterval}` from `@/lib/stripe/prices` (no longer imports `PRICES`)
- Body shape is now `{tier, interval}` — old `{interval}`-only callers get `400 {error:'unknown_tier'}` because `tier` is missing
- Branding tier short-circuits with `400 {error:'use_consult_link'}` and creates NO Stripe Customer, NO Stripe Session
- Unknown tier (anything other than `'basic'|'widget'|'branding'`) returns `400 {error:'unknown_tier'}`
- Valid `(tier, interval)` resolves the Price ID via `getPriceId(tier, interval)` — no direct `PRICES[...]` access
- Section numbering rewritten so account fetch is now Step 2 (was Step 3) and body parsing is Step 3 (was Step 2) — reflects the corrected validation order
- Docstring response-shapes table updated to enumerate the 2 new 400 error codes alongside the existing ones
- Session `metadata` now carries `{accountId, tier, interval}` (was missing `metadata` entirely)
- Log lines for the success path and the stripe-error path now include `tier` alongside `interval` for incident-response observability
- Repo typecheck failures reduced from 5 to 4 (the remaining 4 are all in `app/(shell)/app/billing/page.tsx`, owned by Plan 42.5-05 — exactly as STATE.md predicted)

## Request / response contract

| Body | Status | Response | Stripe side-effect |
|------|--------|----------|---------------------|
| `{tier:'basic', interval:'monthly'}` | 200 | `{url: '<checkout.stripe.com URL>'}` | Customer upserted, Session created with Basic-Monthly Price ID |
| `{tier:'basic', interval:'annual'}` | 200 | `{url: ...}` | Session with Basic-Annual Price ID |
| `{tier:'widget', interval:'monthly'}` | 200 | `{url: ...}` | Session with Widget-Monthly Price ID |
| `{tier:'widget', interval:'annual'}` | 200 | `{url: ...}` | Session with Widget-Annual Price ID |
| `{tier:'branding', ...}` | 400 | `{error:'use_consult_link'}` | None (no Customer, no Session) |
| `{tier:'foo', ...}` | 400 | `{error:'unknown_tier'}` | None |
| `{interval:'monthly'}` (no tier) | 400 | `{error:'unknown_tier'}` | None |
| `{}` (empty body) | 400 | `{error:'unknown_tier'}` | None |
| Any body, no auth cookie | 401 | `{error:'unauthorized'}` | None — auth fires BEFORE tier validation (Phase 41 invariant preserved) |
| Any body, authed but no `accounts` row | 404 | `{error:'account_not_found'}` | None — account fetch fires BEFORE tier validation per plan must_have |

## Validation order (in route)

1. **Auth gate** — `supabase.auth.getClaims()` → 401 `unauthorized` if no claims
2. **Account fetch** — RLS-scoped `accounts` lookup → 404 `account_not_found` if no row
3. **Branding short-circuit** — `tier === 'branding'` → 400 `use_consult_link` (LD-16 — UI should never POST this)
4. **Unknown-tier guard** — `tier !== 'basic' && tier !== 'widget'` → 400 `unknown_tier`
5. **Placeholder-price guard (production only)** — `priceId.startsWith('price_placeholder_')` → 400 `missing_price_id`
6. **Stripe Customer upsert** — race-safe via conditional update on `stripe_customer_id IS NULL`
7. **Stripe Checkout Session create** — with `line_items[0].price = priceId`, `metadata = {accountId, tier, interval}`
8. **Return** — `{url: session.url}`
