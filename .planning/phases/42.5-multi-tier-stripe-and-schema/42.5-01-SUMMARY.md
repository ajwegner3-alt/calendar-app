## Accomplishments

- Forward migration adds `plan_tier TEXT` to `public.accounts` with the LD-14 CHECK constraint and a documentation COMMENT.
- Rollback migration drops the column cleanly via `DROP COLUMN IF EXISTS`.
- Migration is trigger-compatible: confirmed by reading `provision_account_for_new_user` source in `20260510120000_phase41_stripe_billing_foundation.sql` lines 68-98 — the trigger inserts only Phase 10 + Phase 41 columns, never `plan_tier`.
