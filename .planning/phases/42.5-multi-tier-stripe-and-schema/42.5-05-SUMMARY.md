## Decisions Made

- **Default toggle state = "annual"** — carried from legacy `PlanSelectionCard.tsx` line 53 (`useState<"monthly" | "annual">("annual")`). Annual is the savings-headline tab and matches the documented CONTEXT decision.
- **Single shared "Save up to N%" badge in toggle** instead of per-card savings badges — uses `Math.max(basicSavingsPct, widgetSavingsPct)` so the highest savings is advertised. Cleaner visual than two competing badges and avoids per-card visual noise.
- **Container width bumped to `max-w-5xl` (1024px) for 3-card states only** — see Deviations Rule 1 below for full rationale.
- **Comment-only edit to `billing-state-views.tsx`** included in Task 2 to satisfy the strict-grep "zero `PlanSelectionCard` matches in app/" verification gate. The stale comment did not affect runtime but the plan asked for zero matches and a clean grep is easier to verify than "zero except in comments."
- **`use_consult_link` collapsed at the client** rather than relying on Branding never reaching this card. Defense-in-depth — if a future bug routed Branding through `<SubscribeTierCard>`, the user would see the generic "Could not start checkout" copy instead of an enum-string leak.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Container width too narrow for 3-card grid**

- **Found during:** Task 2 (page.tsx refactor)
- **Issue:** The legacy `<div className="container mx-auto max-w-2xl py-8 space-y-6">` constrained content to 672px. With `md:grid-cols-3` rendering inside that 672px box, the cards would be ~210px wide each at desktop — far too narrow to satisfy the verification gate "1024px viewport: all 3 cards visible without horizontal scroll" (the cards would technically render but the gate's intent is comfortable desktop layout, and at 672px the cards would be cramped).
- **Fix:** Bumped `max-w-2xl` → `max-w-5xl` (1024px) for the `plan_selection` and `locked` branches only. Polling and active branches stay at `max-w-2xl` since they render a single-element view (`<CheckoutReturnPoller />` or `<ActiveView />`).
- **Files modified:** `app/(shell)/app/billing/page.tsx` (lines 158, 173)
- **Verification:** `npm run build` succeeds; `/app/billing` route compiles. Visual confirmation deferred to UAT 42.5-06 (no dev server in agent context per plan note).
- **Committed in:** `a7db58c` (Task 2 commit)
