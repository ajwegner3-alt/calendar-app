## Decisions Made

(See `key-decisions` in frontmatter for the full list.)

Most consequential:
- **`priceIdToTier` returns `null`, not throw.** Webhook callsite (Plan 42.5-04) will branch on null and skip the `plan_tier` write rather than failing the Stripe event handler. Throwing would risk Stripe retrying the same event indefinitely.
- **All 4 placeholder strings are distinct** (`price_placeholder_basic_monthly`, `_basic_annual`, `_widget_monthly`, `_widget_annual`). If two were identical, the `PRICE_ID_TO_TIER` object literal would silently alias and the dev environment would write the wrong tier.

## Intentional Typecheck Breakage (Important)

**This plan intentionally leaves the repo's typecheck broken at 5 callsites.** The new `PRICES.basic.monthly` shape is incompatible with the old `PRICES.monthly` shape, so any code that destructures `PRICES.monthly` or `PRICES.annual` directly will fail typecheck. The breakage is confined to:

- `app/(shell)/app/billing/page.tsx` — 4 errors (`PRICES.monthly` / `PRICES.annual` references) — fixed by **Plan 42.5-05**
- `app/api/stripe/checkout/route.ts` — 1 error (`PRICES[interval].priceId` index expression) — fixed by **Plan 42.5-03**

The webhook (`app/api/stripe/webhook/route.ts`) does not currently reference the old shape directly and so does not appear in this typecheck error list — Plan 42.5-04 will add the new `priceIdToTier`-based derivation logic.

This is the documented contract of the plan and matches the wave decomposition: Wave 1 (this plan) ships the new shape in isolation; Wave 2 (plans 03/04/05) updates the consumers. Repo-wide green typecheck returns when Wave 2 completes.
