Constraints to honor (from RESEARCH.md):
- `ADD COLUMN IF NOT EXISTS` — idempotent, safe to re-apply if migration registry is out of sync (see STATE.md tech debt — Phases 36/37/41 migration files exist but not registered in `schema_migrations`; Phase 44 inherits the same risk).
- `BOOLEAN NOT NULL DEFAULT FALSE` — no backfill UPDATE needed; PostgreSQL's `ADD COLUMN` with a constant DEFAULT applies the default to all existing rows automatically (Postgres 11+). All 5 existing v1.7 grandfathered accounts plus any post-Phase-42.5 accounts will get `false` automatically.
- DO NOT modify the trigger function `provision_account_for_new_user` — DEFAULT FALSE means INSERT statements that don't list `cancel_at_period_end` still work.
- DO NOT add an RLS policy change — webhook uses service role and bypasses RLS; billing page reads through the existing accounts RLS policy.
- DO NOT include a separate UPDATE statement for backfill — `DEFAULT FALSE` does this in a single atomic ADD COLUMN.
