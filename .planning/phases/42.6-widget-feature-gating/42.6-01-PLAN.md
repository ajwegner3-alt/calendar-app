<task type="auto">
  <name>Task 1: Create requireWidgetTier helper</name>
  <files>lib/stripe/widget-gate.ts</files>
  <action>
Create `lib/stripe/widget-gate.ts` as a pure function module (no DB, no network, no async).

Required exports:
1. Type `AccountGateInput` (interface) with two fields:
   - `plan_tier: PriceTier | null` (import `PriceTier` from `./prices`)
   - `subscription_status: string | null`
2. Type `WidgetGateResult` (discriminated union):
   - `{ allowed: true }`
   - `{ allowed: false; reason: 'basic_tier' | 'no_subscription' }`
3. Function `requireWidgetTier(account: AccountGateInput): WidgetGateResult`

Logic (in THIS order — order matters for trialing-with-NULL-plan_tier edge case):
1. If `subscription_status === 'trialing'` → return `{ allowed: true }`. (Trialing always allowed regardless of plan_tier, per LD-15 + webhook comment "plan_tier remains NULL during trial — gating treats NULL as trialing — allowed access".)
2. If `plan_tier === 'widget'` AND `subscription_status` is `'active'` OR `'past_due'` → return `{ allowed: true }`. (LD-08 mirror — past_due is banner-only, not lockout.)
3. If `plan_tier === 'basic'` AND `subscription_status` is `'active'` OR `'past_due'` → return `{ allowed: false, reason: 'basic_tier' }`.
4. Fallthrough (everything else: canceled, unpaid, incomplete, NULL subscription_status without trialing, NULL plan_tier without trialing) → return `{ allowed: false, reason: 'no_subscription' }`.

DO NOT:
- Throw on unknown statuses — fall through to no_subscription.
- Log inside the helper — callers may log if they want.
- Add `displayMessage` or other UX-shaped fields — caller renders UX.
- Add `"server-only"` import — function is pure and safe to import anywhere. (`PriceTier` itself comes from a `"server-only"` module, so practically this helper will only be importable server-side anyway. That's fine.)

Wrong illustrative variable name in RESEARCH.md (`status` should be `subscription_status`) — implement correctly here.

File should be ~30-40 lines with JSDoc comments on the exported function explaining each branch and citing LD-04, LD-08, LD-15.

Create `tests/widget-gate.test.ts` following the project's vitest pattern (see `tests/upgrade-action.test.ts` for header style).

File header:
```ts
// @vitest-environment node
import { describe, it, expect } from "vitest";
import { requireWidgetTier } from "@/lib/stripe/widget-gate";
```

Required test cases (one `it()` per case, descriptive names):

ALLOWED branches:
1. `trialing + plan_tier: null` → `{ allowed: true }` (new account edge case — LD-15)
2. `trialing + plan_tier: 'basic'` → `{ allowed: true }` (trialing wins over basic)
3. `trialing + plan_tier: 'widget'` → `{ allowed: true }`
4. `active + plan_tier: 'widget'` → `{ allowed: true }` (canonical paid widget)
5. `past_due + plan_tier: 'widget'` → `{ allowed: true }` (LD-08 mirror — past_due is not lockout)

GATED — basic_tier reason:
6. `active + plan_tier: 'basic'` → `{ allowed: false, reason: 'basic_tier' }`
7. `past_due + plan_tier: 'basic'` → `{ allowed: false, reason: 'basic_tier' }`

GATED — no_subscription reason:
8. `canceled + plan_tier: 'widget'` → `{ allowed: false, reason: 'no_subscription' }` (downgraded post-cancel)
9. `canceled + plan_tier: 'basic'` → `{ allowed: false, reason: 'no_subscription' }`
10. `unpaid + plan_tier: 'widget'` → `{ allowed: false, reason: 'no_subscription' }`
11. `null subscription_status + null plan_tier` → `{ allowed: false, reason: 'no_subscription' }` (account never started subscribing AND never trialed — unlikely but defensive)
12. `incomplete + plan_tier: null` → `{ allowed: false, reason: 'no_subscription' }`

Use `describe('requireWidgetTier', () => { ... })` to group. Each `it()` builds an input literal inline — no shared fixtures needed for a pure function.

Use `expect(result).toEqual({ allowed: true })` and `expect(result).toEqual({ allowed: false, reason: 'basic_tier' })` for assertions (toEqual handles the discriminated union cleanly).
