# Phase 42.6 Plan 02: Public Embed Gate Summary

## Invariants preserved

1. **LD-07 — bare booker never gated.** `git diff --stat 61b65ba..HEAD app/[account]/[event-slug]/page.tsx` → empty (zero lines changed). Bare booker continues to ignore `plan_tier`/`subscription_status` fields even though the shared loader now selects them.
2. **No gate call in the shared loader.** `grep -c "requireWidgetTier" app/[account]/[event-slug]/_lib/load-event-type.ts` → **0**. Gate stays at the route level so each surface (`/embed/*` vs bare booker) owns its own UX.
3. **HTTP 200 + message on gate path — NEVER `notFound()` or `redirect()`.** The `if (!gate.allowed)` branch returns JSX directly; the only `notFound()` call in the file is the pre-existing one for missing data, untouched. Iframe hosts will render the neutral message instead of a browser's X-frame error.
4. **No `force-dynamic` / `revalidate` directives added.** `grep -cE "(force-static|revalidate|force-dynamic)" app/embed/[account]/[event-slug]/page.tsx` → **0**. Route relies on Next's natural dynamic inference (it already awaits `params` + `searchParams` and uses a service-role Supabase client per request).
5. **`proxy.ts` unchanged.** No diff vs `61b65ba`. The existing `Content-Security-Policy: frame-ancestors *` header is already correct for the gated path — the iframe is permitted; only the body content swaps.
