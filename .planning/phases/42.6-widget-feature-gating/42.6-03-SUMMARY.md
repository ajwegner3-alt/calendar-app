## Gate computation (page.tsx)

```ts
import { requireWidgetTier } from "@/lib/stripe/widget-gate";

// account fetched with .select("slug, plan_tier, subscription_status")
let isWidgetAllowed = false;
if (ids.length > 0) {
  const { data: account } = await supabase
    .from("accounts")
    .select("slug, plan_tier, subscription_status")
    .eq("id", ids[0])
    .maybeSingle();
  if (account?.slug) accountSlug = account.slug;
  isWidgetAllowed = account
    ? requireWidgetTier({
        plan_tier: (account.plan_tier ?? null) as "basic" | "widget" | null,
        subscription_status: account.subscription_status ?? null,
      }).allowed
    : false;
}
```

Pass-through to `<EventTypesTable>` as new `isWidgetAllowed` prop alongside existing `accountSlug` + `appUrl`.

## Dialog body branching (embed-code-dialog.tsx)

When `isWidgetAllowed === true`: existing two-column grid with `<EmbedTabs>` + preview iframe pointing at `${appUrl}/embed/${accountSlug}/${eventSlug}`. No behavior change from prior implementation.

When `isWidgetAllowed === false`: replaced with a single `<Card>`:

- Title: "Widget requires an upgrade"
- Body: 3-bullet feature list
  - Embed the booking widget on any website
  - Auto-resizing via script or iframe fallback
  - Brand colors applied inside the iframe
- Footer: full-width `<Button asChild>` wrapping `<Link href="/app/billing">Upgrade to Widget</Link>`

Dialog header also branches:
- Title: `Embed: ${eventName}` (allowed) vs `Widget feature` (gated)
- Description: existing two-snippet pitch (allowed) vs "Upgrade to embed the booking widget on your website." (gated)
